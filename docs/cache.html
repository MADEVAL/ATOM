Cache ♻

PSR‑inspired cache abstraction · Array & File drivers · Remember helpers · Atomic file writes

Introduction

Atom provides a simple cache abstraction via the Driver interface. Two drivers ship with the framework: ArrayDriver for in‑memory, per‑request caching and FileDriver for persistent, disk‑backed caching. The Cache facade wraps the driver and adds convenience methods like remember, rememberForever, increment, and decrement.

Driver interface

get, set(key,value,ttl), has, delete, flush

ArrayDriver

In‑memory hash map, TTL via timestamp check. Zero I/O.

FileDriver

sha1‑keyed files, TTL in first line, atomic tmp+rename, probabilistic GC

Cache facade

get/set/has/delete/flush + remember, rememberForever, increment, decrement

Usage

Obtaining the cache instance

// Lazy, config‑driven — uses APP_CACHE_DRIVER env var
$cache = $app->cache();

The cache instance is lazily created on first access. The driver is selected based on APP_CACHE_DRIVER in the environment: 'file' (default, uses cacheDir) or 'array'. When no cacheDir is configured, the file driver falls back to sys_get_temp_dir()/atom/cache.

Basic operations

$cache->set('user:42', ['name' => 'Alice'], ttl: 3600);

if ($cache->has('user:42')) {
    $user = $cache->get('user:42');
}

$value = $cache->get('missing', 'default value');

$cache->delete('user:42');
$cache->flush();  // clear everything

Remember pattern

// Fetch from cache or compute + store
$stats = $cache->remember('dashboard_stats', function() use ($db) {
    return $db->all('SELECT ...');
}, ttl: 600);

// Remember forever (no TTL expiry)
$config = $cache->rememberForever('app_config', fn() => loadConfig());

Counters

$cache->increment('page_views');              // +1
$cache->increment('batch_size', 100);        // +100
$cache->decrement('stock');                   // -1
$cache->decrement('quota', 5);               // -5

Drivers

ArrayDriver

In‑memory cache stored in a PHP array. TTL is enforced via timestamp comparison on every get() and has() call — expired entries are automatically deleted. Best for ephemeral, per‑request data or unit testing.

use Atom\Cache\{Cache, ArrayDriver};

$cache = new Cache(new ArrayDriver());
$cache->set('temp', 'data', ttl: 60);

FileDriver

Persistent, disk‑backed driver. Each cache entry is stored as a PHP file at {dir}/sha1(key).php. The first line holds the expiry timestamp (0 = no expiry), followed by a newline and the serialized value.

use Atom\Cache\{Cache, FileDriver};

$cache = new Cache(new FileDriver(
    dir: __DIR__ . '/storage/cache',
    cleanupChance: 100,  // 1 in 100 chance of GC on set()
));

$cache->set('key', 'value', ttl: 3600);

Atomic writes: data is written to a random temp file first (.tmp suffix with 8 random hex bytes), then rename()d into place with LOCK_EX. This prevents partial reads during concurrent writes.

Garbage collection: expired files are cleaned up probabilistically — on each set() there is a 1/cleanupChance probability that all files in the cache directory are scanned for expired entries. Additionally, expired keys return null and are deleted on read.

Configuration

Env variableDefaultDescription
APP_CACHE_DRIVERfileDriver: 'file' or 'array'

The driver is selected in Application::cache() based on APP_CACHE_DRIVER. For 'file', the directory is taken from Config::$cacheDir. No additional setup is required — calling $app->cache() initializes everything on first use.
