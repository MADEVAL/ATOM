Introduction

Atom's ORM provides an active‑record style abstraction over the Database layer. Models map to database tables, public properties map to columns, and PHP 8 attributes control metadata. The query builder chains where, orderBy, limit, count, and more — all with parameterized queries.

Model

Active‑record: create, find, save, delete, hydrateRow, fill, toArray

Query

Fluent builder: where, whereIn, orWhere, whereBetween, orderBy, limit, paginate

Relations

hasMany, belongsTo, hasOne — with eager loading via with()

Metadata

Auto‑detected from attributes & property types; cached per class

Setup

use Atom\Database\Database;
use Atom\Orm\Model;

$db = new Database('sqlite:/path/to/db.sqlite');
Model::setConnection($db);

All models share the connection set via Model::setConnection(). Calls to Model::db() (from any model) will throw RuntimeException if no connection has been set.

Model definition

use Atom\Orm\{Model, Table, PrimaryKey, Column};

#[Table('users')]
class User extends Model {
    #[PrimaryKey]
    public int $id = 0;

    #[Column('user_name')]
    public string $name = '';

    public string $email = '';
    public string $created_at = '';
    public string $updated_at = '';
}

Attributes

AttributeDescription
#[Table('name')]Set the database table name. If omitted, derived from class short name (e.g. UserProfile → user_profiles).
#[PrimaryKey]Mark a property as the primary key column. Defaults to id.
#[Column('col_name')]Map a PHP property to a different database column name.

Only public non‑static properties are mapped to columns. Metadata is cached per model class (one reflection pass).

Type casting

Public property types (int, float, bool, string) are automatically cast when filling from arrays (fill()) and when hydrating from database rows (hydrateRow()).

#[Table('posts')]
class Post extends Model {
    #[PrimaryKey]
    public int $id = 0;
    public string $title = '';
    public bool $published = false;
    public float $rating = 0.0;
    public int $user_id = 0;
}

Timestamps

If a model has public properties named created_at and/or updated_at (matched by column name), they are automatically populated with date('Y-m-d H:i:s') on insert and update. Disable with protected bool $timestamps = false on the model class.

CRUD operations

Creating records

// Create and save in one call
$user = User::create(['name' => 'Alice', 'email' => 'alice@ex.com']);

// Create manually
$user = new User();
$user->fill(['name' => 'Bob', 'email' => 'bob@ex.com']);
$user->save();

// First or create — find by criteria or insert
$user = User::firstOrCreate(
    ['email' => 'alice@ex.com'],               // search
    ['name' => 'Alice'],                         // merge on insert
);

Reading records

// Find by primary key — returns null if not found
$user = User::find(1);

// Find or throw RuntimeException
$user = User::findOrFail(42);

// Check if loaded from database
$user->exists();  // bool

Updating records

$user = User::find(1);
$user->name = 'Alice Updated';
$user->save();

// Batch fill + save
$user->fill(['name' => 'New Name', 'email' => 'new@ex.com']);
$user->save();

Deleting records

// Delete loaded model
$user = User::find(1);
$user->delete();

// Destroy by primary key(s) — returns count
$deleted = User::destroy(1, 2, 3);  // int

Converting to array

$user = User::find(1);
$data = $user->toArray();
// → ['name' => 'Alice', 'email' => 'alice@ex.com', ...]
// PK is excluded from toArray() output

Query builder

// Start a fluent query
$query = User::query();

// WHERE clauses
$query->where('active', 1);                          // column = value
$query->where('age', '>', 18);                        // column > value
$query->whereIn('role', ['admin', 'user']);
$query->whereNotIn('status', ['deleted', 'banned']);
$query->whereNull('deleted_at');
$query->whereNotNull('email');
$query->whereBetween('price', 10, 100);
$query->orWhere('role', 'guest');

// Order, limit, offset
$query->orderBy('created_at', 'DESC');
$query->orderByDesc('id');
$query->limit(10);
$query->offset(20);

// Execute
$users = $query->get();                 // list of Model instances
$user  = $query->first();             // first result or null
$total = $query->count();             // int — COUNT(*)
$exists = $query->exists();           // bool

Magic whereX methods

// whereRole('admin')  →  WHERE "role" = 'admin'
// whereStatus('active') → WHERE "status" = 'active'
$admins = User::query()->whereRole('admin')->get();
$active = User::query()->whereIsActive(1)->get();

// CamelCase is auto‑snake_cased for the column name
$users = User::query()->whereCreatedAt('2025-01-01', '>', 0)->get();
// auto adjusts: whereCreatedAt('value') → "created_at" = ?
//              : whereCreatedAt('value', '>') → "created_at" > ?

Pagination

// Query builder paginate — returns Paginator instance
$pager = Post::query()
    ->where('published', true)
    ->orderBy('created_at', 'DESC')
    ->paginate(perPage: 20);

// Or pass a Request for ?page=&per_page= query params
$pager = Post::query()->paginate(20, $request);

// Paginator provides structured output
$result = $pager->toArray();
// → ['data' => [...], 'page' => 1, 'perPage' => 20, 'total' => 150, 'pages' => 8]

Relations

Defining relations

#[Table('posts')]
class Post extends Model {
    #[PrimaryKey] public int $id = 0;
    public string $title = '';
    public int $user_id = 0;

    public function author(): BelongsTo {
        return $this->belongsTo(User::class);
    }

    public function comments(): HasMany {
        return $this->hasMany(Comment::class);
    }
}

Relation types

MethodForeign key defaultReturns
$this->hasMany(Related::class, ?$fk, ?$lk){this_table}_id (e.g. post_id)Collection of related models
$this->belongsTo(Parent::class, ?$col, ?$ok){parent_table}_id (e.g. user_id)Single parent model
$this->hasOne(Related::class, ?$fk, ?$lk){this_table}_idSingle related model

All foreign key / local key parameters are optional and auto‑derived from the table names. You can override them explicitly:

$this->hasMany(Comment::class, foreignKey: 'post_fk', localKey: 'id');
$this->belongsTo(User::class, column: 'author_id', ownerKey: 'id');

Eager loading

// Eager load prevents N+1 queries
$posts = Post::query()
    ->with('author', 'comments')
    ->get();

foreach ($posts as $post) {
    echo $post->author->name;       // already loaded
    foreach ($post->comments as $c) {  // already loaded
        echo $c->body;
    }
}

Eager loading works as follows: relation methods on the first model are introspected to determine foreign/local keys, a single SELECT ... WHERE fk IN (...) query fetches all related records, and results are distributed to parent models by key mapping.

Advanced

Connection

// Set once before using any model
Model::setConnection($db);

// Connection is shared across all models
protected static function db(): Database {
    return self::$connection
        ?? throw new RuntimeException('ORM connection not set.');
}

Custom table name and primary key

#[Table('blog_posts')]
class Post extends Model {
    #[PrimaryKey] public string $uuid = '';
}

class FlightBooking extends Model {
    // Auto table name: flight_bookings
    // snake_case: UserProfile → user_profiles
    // pluralised: adds trailing 's'
}

SQLite insert and lastInsertId

$post = Post::create(['title' => 'Hello']);
// $post->id populated via Database::lastId()
// Auto‑detected: PK excluded from INSERT when value is 0 or null
